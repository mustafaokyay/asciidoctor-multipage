About This Sample

1. Tree Nodes

The MultipageHtml5Converter splits the document into node types during processing, based on the value of the multipage-level attribute. If a node’s level is equal to the multipage-level value specified for it or the nearest ancestor node, then the node is generated as a leaf node. In this context, "node" refers to an Asciidoctor Document or Section.

For this document, the following node types are determined. A file is written for each root, branch, and leaf node. Each content node is included in its parent (leaf) node file.

MultipageHtml5Converter Sample – multipage-level=0, source document root → root node

  • About This Sample – level-0 part → leaf node

    1. Tree Nodes – level-1 chapter → content node

    2. Output Files – level-1 chapter → content node

  • Custom Level Part – multipage-level=1, level-0 part → branch node

    1. One Chapter – level-1 chapter → leaf node

      1. Cross-References – level-2 section → content node

    2. Another Chapter – level-1 chapter → leaf node

      1. More Cross-References – level-2 section → content node

2. Output Files

The MultipageHtml5Converter creates the following five output files from this sample.adoc document.

MultipageHtml5Converter Sample – root node with id/title [[sample]] (from the file name) → the main output file (sample.html by default)

  • About This Sample – leaf node with id [[about-sample]] → about-sample.html

    1. Tree Nodes – content node → included in parent node

    2. Output Files – content node → included in parent node

  • Custom Level Part – branch node with id [[custom-level]] → custom-level.html

    1. One Chapter – leaf node with id [[custom-chapter-1]] → custom-chapter-1.html

      1. Cross-References – content node → included in parent node

    2. Another Chapter – leaf node with id [[custom-chapter-2]] → custom-chapter-2.html

      1. More Cross-References – content node → included in parent node
